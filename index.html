Overview

Abstract

Instruction-guided 3D editing has recently progressed from optimization-based pipelines to native 3D foundation models. However, existing benchmarks rarely provide explicit annotations of edited regions or standardized evaluation protocols. It therefore remains difficult to systematically assess where an edit occurs, whether the intended transformation is faithfully realized, and how well unedited content is preserved.

We present Omni3DEdit, a unified benchmark for instruction-guided 3D editing with explicit edited-region annotations. Omni3DEdit contains 128,906 paired source–target 3D assets spanning five subsets that cover pose changes, rigid part addition and removal, articulated joint-state changes, region-targeted generative modifications, and material edits. Each sample provides a source asset, a target asset, a natural-language instruction, an edited-region annotation, lightweight edit metadata, and provenance information.

The benchmark is standardized into a common GLB-based representation and supports object-centric evaluation of instruction faithfulness, preservation outside the edited region, edit locality, and subset-specific validity. We further provide a practical benchmark protocol with strong off-the-shelf baselines and locality-aware metrics.

Data

Dataset

Five diverse edit families with explicit edited-region annotations

Pose

23,648 pairs

Pose changes derived from animated or rigged assets in Objaverse. Representative frames selected via pairwise deformation maximization.

Objaverse

Structure

40,000 pairs

Programmatically constructed part addition/deletion pairs from hierarchical part annotations. Bidirectional edits with exact region annotation.

PartNet

Articulation

6,875 pairs

Kinematic state changes by modifying URDF joint parameters. Edited region captures the volumetric sweep of articulating components.

PartNet-Mobility

Part-Edit

26,407 pairs

Part-targeted local modifications via VoxHammer. Diverse instructions and realistic free-form edits with explicit region isolation.

PartNet VoxHammer

Material

31,976 pairs

Appearance-only edits modifying texture or PBR material while preserving geometry. Atomic single-part material changes.

3DCoMPaT++

Standardization Pipeline

All assets are converted into a unified GLB-based representation with standardized coordinate conventions, mesh normalization, multi-view rendering, and quality control.

Unified Annotation Schema

Field Description
sample_id Unique sample identifier
source_asset, target_asset File paths or identifiers for source and target GLB models
instruction Natural-language edit instruction
edited_region Mesh-space edited region and projected view masks for locality / preservation evaluation
edit_operation Edit operation type (add, remove, open, rotate, deform, recolor, restyle, …)
category Object category or asset group
provenance Source dataset, generation pipeline, redistribution status, and licensing metadata
Evaluation

Benchmark Protocol

Region-aware evaluation along four complementary dimensions

Instruction Faithfulness

CLIP-T between multi-view renderings of the edited prediction and the input instruction. DINO-I or CLIP-I similarity when ground-truth target is available.

CLIP-T ↑ DINO-I ↑ CLIP-I ↑

Preservation

Geometry and render metrics on the complement of the edited region: preserved-region Chamfer Distance, masked PSNR, SSIM, and LPIPS.

CD ↓ mPSNR ↑ mSSIM ↑ mLPIPS ↓

Edit Locality

Compare predicted change support with annotated edited region via binary change maps. Region-level precision, recall, and F1.

Precision ↑ Recall ↑ Region-F1 ↑

Subset-Specific Validity

Preservation and locality metrics depend only on the non-edited region. The benchmark supports both text-guided and image-conditioned evaluation.
